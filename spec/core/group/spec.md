# Group（グループ管理）仕様

## 概要

ユーザーをグループとして組織化し、グループ単位での予定管理・空き計算の基盤を提供するコアモジュール。

## 機能要件

### メンバーロール

| ロール | 権限 |
|--------|------|
| owner | グループの全管理権限（削除・メンバー管理・予定管理） |
| admin | メンバー管理・予定管理 |
| member | 閲覧・自身の予定追加 |

### 予定の種類

| 種別 | ベース | 用途 |
|------|--------|------|
| groupSchedules | 曜日ベース (recurring / oneshot) | 毎週の定期予定 |
| groupEvents | 日付ベース | 学校行事・試験・休校日 |

### 振る舞い規則

- ユーザーは複数グループに所属可能
- グループ予定（groupSchedules）は曜日ベースの繰り返し予定
- グループイベント（groupEvents）は日付ベースの個別予定
- グループ予定はメンバー全員の空き計算に影響する
- ラベルを付与することでターム単位の一括管理が可能

### モジュール選択機能

- グループごとに有効化するモジュールを選択可能 (`enabledModules`)
- 選択可能モジュール: CALICULA, PM, MACHINA, Notification, Voting, Holiday 等
