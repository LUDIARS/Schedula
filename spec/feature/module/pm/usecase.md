# PM ユースケース

## UC-PM-01: プロジェクト作成

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. プロジェクト名・ソース種別（github / notion）を入力する
  2. 接続設定（owner, repo, token 等）を入力する
  3. 同期間隔を設定する
  4. プロジェクトを作成する

## UC-PM-02: タスク同期（Pull）

- **アクター**: システム / ユーザー
- **主フロー**:
  1. 外部ソースからタスク一覧を取得する
  2. ローカルタスクとの差分を検出する
  3. 新規タスクは作成、変更タスクは更新する
  4. スナップショットを記録する

## UC-PM-03: タスク同期（Push）

- **アクター**: ユーザー
- **主フロー**:
  1. ローカルで変更されたタスク（dirty_flag）を検出する
  2. 外部ソースに変更を書き戻す
  3. dirty_flag をクリアする

## UC-PM-04: コンフリクト解決

- **アクター**: ユーザー / システム
- **主フロー**:
  1. 同期時にローカル・外部の両方が変更されたタスクを検出する
  2. 解決戦略に基づいてマージする
  3. 手動解決の場合はユーザーに確認を求める

## UC-PM-05: タスク内容検証

- **アクター**: ユーザー
- **主フロー**:
  1. タスクを選択して検証を実行する
  2. 充実度スコア・問題・改善提案を表示する

## UC-PM-06: 分析レポート生成

- **アクター**: ユーザー
- **主フロー**:
  1. プロジェクトを選択する
  2. レポート種別（progress / critical_path / gompertz）を選択する
  3. 分析結果を表示する
