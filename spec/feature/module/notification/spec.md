# Notification（通知）仕様

## 概要

Webhook・Bot によるマルチチャンネル通知を提供する M5 モジュール。イベント駆動アーキテクチャで、予約・投票・スケジュール変更などのイベントに応じて通知を配信する。

## 機能要件

### 配信チャンネル

| チャンネル | 送信方法 |
|-----------|---------|
| Slack | Webhook / Bot |
| Discord | Webhook / Bot |
| LINE | LINE Notify |
| Generic | カスタム Webhook |

### イベントバス

- インメモリのイベントバスで非同期イベント配信
- イベント名: `reservation.created`, `voting.created`, `schedule.changed` 等

### テンプレートエンジン

- `{variable}` 形式の変数置換
- コードブロックフォーマット対応
- プラットフォーム別テンプレートサポート

### 通知設定

- ユーザーごとのチャンネル別通知設定
- イベントフィルタリング
- リマインダー設定（前日・当日朝・直前）
- quiet hours（通知停止時間帯）

### Webhook 配信

- HMAC 署名による認証
- リトライロジック（失敗時の再配信）
- 配信ログの記録
