# M1: 時間割自動生成 仕様

## 概要

動的計画法（DP）と制約充足問題（CSP）を組み合わせ、カリキュラムを時間割に自動配置するサブモジュール。2つの配置モードと入れ替え候補のハイライト機能を持つ。

## 配置アルゴリズム

### DP による配置手順

1. 各カリキュラムの候補スロット数を計算する
2. 候補が少ない順（MRV: Minimum Remaining Values）にソートする
3. 各カリキュラムを順番に配置し、制約伝播で矛盾を検出する
4. バックトラックで配置不可時に別候補を試行する

### 制約条件

- 講師の出講可能スロット内であること
- 同一講師が同一コマに複数授業を持たないこと
- 同一教室が同一コマに重複使用されないこと
- カリキュラムの必要コマ数を満たすこと

### 配置モード

| モード | 戦略 | ユースケース |
|--------|------|-------------|
| Pack（詰め込み） | 制約の厳しい科目から優先配置。早い曜日・コマに密集 | 教室稼働率最大化、省エネ |
| Spread（分散） | 曜日をまたいで均等配置。ラウンドロビン方式 | 学生の負荷分散、空きコマ確保 |

## 入れ替え候補ハイライト

### 候補数による色分け

| 候補数 | 色 | 意味 |
|--------|-----|------|
| 3コマ以下 | 灰色 (#6E7681) | 候補少 — 入れ替え困難 |
| 4〜14コマ | オレンジ (#D29922) | 候補中 — 慎重に選択 |
| 15コマ以上 | 緑 (#3FB950) | 候補多 — 柔軟性あり |

## 無限ループ防止

- 配置時の候補が15コマ以上の講師の授業は再計算対象外とする
- 入れ替えの連鎖は最大2段階まで（直接入れ替え + 影響先の再配置）
- 各入れ替え操作に Transaction ID を付与し、同一 Transaction 内の再入を禁止する

## 時間割定義

| コマ | 開始 | 終了 |
|------|------|------|
| 1限 | 09:30 | 10:30 |
| 2限 | 10:30 | 11:30 |
| 3限 | 11:30 | 12:30 |
| 4限 | 12:30 | 13:30 |
| 5限 | 13:30 | 14:30 |
| 6限 | 14:30 | 15:30 |
| 7限 | 15:30 | 16:30 |
| 8限 | 16:30 | 17:30 |
| 9限 | 17:30 | 18:30 |
| 10限 | 18:30 | 19:30 |
| 11限 | 19:30 | 20:30 |

曜日: 月〜日（7日間、0=月〜6=日）

## エッジケース

- すべてのカリキュラムを配置できない場合、未配置リストを返す
- 講師が1名しかいない科目は最優先で配置される（MRV）
- 教室が不足する場合、roomId を null として仮配置する
