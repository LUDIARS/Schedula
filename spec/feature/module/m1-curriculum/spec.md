# M1: カリキュラム管理 仕様

## 概要

学科・講師・カリキュラム・ターム・出講可能スロットを管理するサブモジュール。時間割自動生成の入力データを構築する。教務管理者（admin）専用。

## 機能要件

### エンティティ

- **学科 (departments)**: トップレイヤの設定項目。カリキュラムの所属先
- **講師 (instructors)**: トップレイヤの設定項目。カリキュラムの担当者
- **カリキュラム (curricula)**: 学科に所属する授業科目。講師・コマ数・タームを持つ
- **ターム (terms)**: カリキュラムの期間区分（例: 前期、後期）
- **出講可能スロット (instructorAvailableSlots)**: 講師ごとの出講可能な曜日・コマ

### 学科合同授業

- 1カリキュラムが複数の学科に所属可能（中間テーブル `curriculumDepartments`）
- 合同授業は配置時に全学科の制約を考慮する

### 教室管理

- 教室タイプ: 講義室 / 演習室 / PC室 / 実験室 / 大講義室
- 設備リスト（JSON配列）: プロジェクター、ホワイトボード等

### CSV インポート

- **講師CSV**: `名前,専攻,担当科目1,...,月1,月2,...,日11` (空きマトリクス: 1=空き, 0=不可)
- **教室CSV**: `教室ID,教室名,定員,タイプ,設備1,...`
- **カリキュラムCSV**: `科目名,専攻,担当講師,週コマ数,教室タイプ`

### エッジケース

- 講師が未アサインのカリキュラムは自動配置の対象外
- タームが未設定のカリキュラムはデフォルトターム扱い
- CSV インポート時の重複は上書き or スキップを選択可能
