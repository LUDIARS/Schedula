# Auth ユースケース

## UC-AUTH-01: ユーザー登録

- **アクター**: 未認証ユーザー
- **事前条件**: なし
- **主フロー**:
  1. ユーザーがメールアドレス・名前・パスワードを入力する
  2. システムがメールアドレスの重複を検証する
  3. パスワードをハッシュ化して保存する
  4. JWT トークンペア（access + refresh）を発行する
- **代替フロー**:
  - メールアドレスが既存の場合、エラーを返す
- **事後条件**: ユーザーが作成され、認証済み状態になる

## UC-AUTH-02: パスワードログイン

- **アクター**: 登録済みユーザー
- **主フロー**:
  1. メールアドレスとパスワードで認証する
  2. bcrypt によるハッシュ照合を行う
  3. JWT トークンペアを発行する
  4. 最終ログイン日時を更新する
- **代替フロー**:
  - 認証失敗時、汎用エラーメッセージを返す（情報漏洩防止）

## UC-AUTH-03: Google OAuth ログイン

- **アクター**: Google アカウント保持ユーザー
- **主フロー**:
  1. Google OAuth 認可フローを開始する
  2. 認可コードをトークンに交換する
  3. Google プロフィール情報を取得する
  4. 既存ユーザーと紐付け、または新規作成する
  5. Google トークン（access / refresh）を保存する
  6. JWT トークンペアを発行する
- **事後条件**: Google Calendar 連携に必要なトークンが保存される

## UC-AUTH-04: トークンリフレッシュ

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. refresh token を送信する
  2. トークンの有効性を検証する
  3. 新しい access token を発行する
- **代替フロー**:
  - refresh token 期限切れの場合、再ログインを要求する

## UC-AUTH-05: ログアウト

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. セッションを無効化する
  2. refresh token を削除する

## UC-AUTH-06: パスワードリセット

- **アクター**: 登録済みユーザー
- **主フロー**:
  1. メールアドレスを入力する
  2. リセットトークンを生成・送信する
  3. トークンを検証し新パスワードを設定する
