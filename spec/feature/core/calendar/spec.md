# Calendar（カレンダー）仕様

## 概要

個人の予定管理と Google Calendar との双方向同期を提供するコアモジュール。Google 認証なしでも手動で予定を登録できる。M2（データ統合）の役割を兼ね、各データソースを統合する。

## 機能要件

### プライバシー規則

| データ種別 | 公開範囲 | 他者への公開内容 |
|-----------|---------|----------------|
| 授業予定 | 専攻学科名のみ | 「情報工学・2限」のように科目名は非公開 |
| 個人カレンダー | 本人のみ | 「予定あり」としてのみ表示 |
| 学校行事 | 全ユーザー | 行事名・日時を公開 |
| 予約情報 | 全ユーザー | タイトル・教室・参加者を公開 |

### 統合スロットモデル

| 状態 | 説明 | ソース |
|------|------|--------|
| free | 空き | — |
| class | 授業 | M1 |
| personal | 個人予定 | 手動入力 / Google Calendar |
| event | 学校行事 | Google Calendar / グループイベント |
| reserved | 予約済み | M4 |

### キャッシュ戦略

- M1データ: 学期単位でキャッシュ、変更時に無効化
- Google Calendar: 15分間隔でポーリング、差分同期
- 個人カレンダー: ユーザーリクエスト時に取得（10分TTL）
- M4フィードバック: イベントドリブンで即時反映
